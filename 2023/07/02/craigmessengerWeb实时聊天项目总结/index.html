CraigMessenger——Web实时聊天项目总结

项目简述：

基于PusherJS库的Web端实时聊天项目

技术栈

React Next.js13 Next-Auth Cloudinary Pusher-js Prisma MongoDB

页面分割

主要有两个页面

一个是登陆页面，另一个是聊天页面 聊天页面由聊天列表和用户列表组成。

聊天页面主要是有侧边栏sidebar和聊天窗口。

主要功能

  1. 登陆

    使用Next-Auth验证，普通账号密码，社交媒体Github以及Google登陆

  2. 实时聊天

    在登陆之后需要进行权限验证，使用Next-Auth中的withAuth方法，他的底层逻辑是检查浏览器的cookie，对于传入的url路径参数，如果能够查找到对应cookie，则更够访问。查找不到则重定向到登陆界面

    整体的实时通信都是用Pusher来实现的，pusher底层是基于WebSocket来实现的。

    他提供了基于事件的API接口，并且易于查询频道情况，在线用户等常用信息。并且提供了端到端的信息加密，保证了信息安全。

    在使用前要现在官网注册账号并申请APPid。在使用时进行配置在服务端进行配置。

    在前端消息显示的Body组件里，首先需要订阅conversationId对应频道。并且绑定对应的消息创建和更新事件，在有更新的时候进行视图的更新。

    同样在sidebar的conversation显示模块，绑定的则是会话的创建，更新与取消，对应有更新时进行试图更新。

    在后端发送消息后，除了将对应信息存入数据库，还会在pusher的channel里触发消息更新创建和会话更新事件。

    还有一个被看到的功能

  3. 使用tailwindcss简化编码，把CSS代码集成到html的classname里，无需额外css文件。并且对于不同显示大小的终端可以进行很好的适配。
